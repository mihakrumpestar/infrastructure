## General guidelines

- follow best coding practices and dry, kiss, yagni principles
- scan the entire codebase before making a change to indentify all places that need changes
- performance, readability and mentability are important
- careful with regressions
- triple check your work
- always use MCP tools to fetch up-to-date information
- do not overengineer
- if you do not understand something or something is strange, ask me or raise concern
- do not remove comments, unless explicitly told to do so

## MCP tools

When you need to search all non-golang docs, use `context7` tools.

When you need to search golang docs, use `godoc` tools.

When you need to search nixos/nix docs, use `nixos` tools.
